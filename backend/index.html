Templates API Documentation

This document provides information about the Templates API, which allows you to manage templates in a database.

Endpoints

1. Create a Template

Method: POST

Endpoint: /templates

Request Body:


{
  "name": "Template Name",
  "category": "Template Category",
  "price": 99.99,
  "template_link": "https://www.example.com/template",
  "short_description": "Short description of the template",
  "long_description": "Detailed description of the template"
}

2. Get All Templates

Method: GET

Endpoint: /templates

3. Get a Template by ID

Method: GET

Endpoint: /templates/{id}

Example: /templates/123

4. Update a Template

Method: PUT

Endpoint: /templates/{id}

Request Body:


{
  "name": "Updated Template Name", 
  "category": "Updated Template Category",
  "price": 129.99,
  "template_link": "https://www.example.com/updated-template",
  "short_description": "Updated short description",
  "long_description": "Updated detailed description"
}

5. Delete a Template

Method: DELETE

Endpoint: /templates/{id}

Example: /templates/123

Data Model

Column Data Type Description
id SERIAL Unique identifier for the template (auto-incrementing)
name VARCHAR(255) Name of the template
category VARCHAR(255) Category the template belongs to
price NUMERIC(10, 2) Price of the template
template_link TEXT URL link to the template
short_description TEXT Brief description of the template
long_description TEXT Detailed description of the template

Error Handling

The API will return standard HTTP status codes to indicate success or failure of a request. Common codes include:

  • 200 OK: Successful request.
  • 201 Created: New resource created successfully.
  • 400 Bad Request: The request was malformed or missing required data.
  • 404 Not Found: The requested resource was not found.
  • 500 Internal Server Error: An unexpected error occurred on the server.
